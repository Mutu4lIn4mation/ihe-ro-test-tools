Registered Contourer Actor - Test Scenario 1 (non-hybrid system)
Instructions for use test scenarios
Module Overview
Description
The Image Registration Profile specifies how images, RT Structure Sets, RT Doses, and associated spatial registration information can be exchanged, stored, processed and displayed.
This scenario will test that the Registered Contourer is able to create RT structure sets consistent with the IHE-RO rules using non-hybrid image data.
The Registered Contourer Actor receives the images (CT, MR and/or PET), Spatial Registration, and related RT Structure Set; each image may have an RT Structure Set created in its Frame of Reference. The Registered Contourer will store the RT Structure Set(s) to the Archive.
The Registered Contourer actor should be able to support the following SOP classes:
·        CT Image Storage
·        MR Image Storage
·        Positron Emission Tomography Image Storage (PET)
·        Spatial Registration Storage
·        RT Structure Set Storage
For this Registered Contourer test scenario DVTk as Archive actor will receive data from the Registered Contourer.
·        Verify that the (resampled) RT Structure Set output of a Registered Contourer is properly formatted, references the correct image data sets, and is correctly aligned.
·        When the scenario starts a dialog will pop up where the user can select the desired scenario flow:
1. Send: CT-PET image series, one spatial registration (CT-PET), one structure set based on CT
·        Receive: one registered structure set that is resampled to the frame of reference of PET
2. Send: CT-MR image series, one Spatial Registration (CT-MR), one structure set based on CT
·        Receive: one registered structure set that is resampled to the frame of reference of MR
3. Send: CT-MR and PET image series, two spatial registrations (CT-MR and CT-PET), one structure set based on CT
·        Receive: one registered structure set that is resampled to the frame of reference of MR or PET
Transactions sent to actor
RAD-4.8: Modality Images Stored
MMRO-II-2: Utilize Spatial Registrations
MMRO-4: Registered Structure Set Retrieval
Expected transactions from actor
MMRO-3: Registered Structure Set Storage
Used scenario dataset
The dataset supplied with the installation of the IHE-RO MMRO II testtool.
