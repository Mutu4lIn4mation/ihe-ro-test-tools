Send Retrieve Input Objects
Instructions for use test scenarios
Send Retrieve Input Objects rule overview
Description:
Test scenario entry point: First execute the Send UPS worklist query test scenario.
1. Make sure the system under test (SUT) is active and listening.
2. Start the scenario. 
3. Select the UPS to use and the test tool sends the C-Move request and starts to listen for incoming C-STORE request(s).
4. Send the C-Move response(s) and C-STORE request(s) to the test tool.
5. The test tool sends the C-Store response(s) .
6. The test tool automatically starts validating the C-Move response and the C-Store request.
Expected transactions from actor:
Validation of transaction:
RO-18: Retrieve Workitem Input Objects from Object Storage
RO-20: Retrieve Dynamic Treatment Delivery Input Objects from Treatment Management System
Extra information:
This scenario contains an extra rule for continuation.
If selected UPS is an continuation treatment UPS and the input information sequence contains an reference to RT Beams Delivery Instrucion the treatment delivery type should be 'CONTINUATION'
