Instructions for use test scenarios
Receive Retrieve Input Objects rule overview
Description:
Test scenario entry point: First execute the Receive UPS Worklist Query – test scenario.
1. Start the scenario, choose which actor to play as test tool (OST, TMS or both).
2.  The test tool starts to listen for incoming messages.
2. Send the C-Move Request with the PDS actor. 
3. The test tool sends C-Move pending responses to the PDS actor and the test tool sends C-Store requests to the PDS actor (based upon the C-MOVE-RQ)
4. The PDS Actor should send C-STORE responses to the test tool.
5. When the DICOM objects are successfully stored, the test tool will send a C-MOVE success response.
6. When the communication is finished, press ‘stop listening and start validation’.
7. The test tool validates the C-Move requests and C-Store responses.

Additional technical information
The test tool will use the dataset folder to check if it can fulfill the incoming C-MOVE request. If the study instance UID, Serie instance UID and the SOP instance UID matches a DICOM object in the datasetfolder, a C-STORE-RQ will be send to the PDS, otherwise an error is generated.
