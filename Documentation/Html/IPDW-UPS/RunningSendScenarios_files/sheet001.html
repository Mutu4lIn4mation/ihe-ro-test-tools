Instruction for use send test scenarios
This document describes how to run the send test scenarios using the DVTk based IHE-RO TestTool.
Start one of the IHE-RO test tools by clicking the TestTool IPDW UPS” shortcut.
There are two DVTk project files, one for the PDS scenario's and one for the TMS scenario's.
Configuration
First the Test Tool needs to be configured, starting with the connection details. In the send scenario's the test tool is playing the PDS actor.
 A Test Tool Configuration Editor is delivered along with the scenarios.
Click here to read how to use the Test Tool Configuration Editor. Note that this configuration is used instead of the session file!
Running a scenario
The test scenario will start running by double clicking on the test scenario name in the left pane (e.g. double click on the text “Send-01-UPS-WorklistQuery.vbs” in the left pane)
For each send scenario be sure that the system under test (SUT) is active and listening on the correct port and IP adress that is configured with the test tool configuration editor.
Example: Send UPS Worklist Query
Start the scenario by double clicking on the text “Send-01-UPS-WorklistQuery.vbs” in the left pane. The following pop-ups will be displayed.
In this scenario the test tool will send a C-Find-Request to the actor under test. In the logging window the outgoing and incoming DICOM messages are shown. 
This scenario will stop automatic after it has received the responses on the sended query. All received responses are validated against the IHE-RO objects profile requirements. 
The results will be shown in the test tools main window.
Test Results
The result files of all tested scenarios are located at “<Instalation dir>\TestTools IPDW-V2\ResultFiles”.
