Registered Dose Display Actor - Test Scenario 2 (Hybrid)
Instructions for use test scenarios
Description
The Image Registration Profile specifies how images, RT Structure Sets, RT Doses, and associated spatial registration information can be exchanged, stored, processed and displayed.
Because the RT Structure Set shares a Frame of Reference UID with one of the datasets, the structures will be transformed by applying the same transformation to the coordinates in the structure set as the dataset.
Because this scenario uses hybrid system data no spatial registration transformation is required; the Registered Dose Display re-samples the datasets, if necessary to match resolution for display.
This scenario will test that the Registered Dose Display consistently handles the input data sets.
The Registered Dose Display actor should be able to support the following SOP classes:
·        CT Image Storage
·        MR Image Storage
·        Positron Emission Tomography Image Storage (PET)
·        RT Dose Storage
·        RT Structure Set Storage
·        RT Plan Storage
For this Registered Dose Display test scenario DVTk as Archive actor will store data to the Registered Dose Display.
·        Determine that the inputs to a Registered Dose Display actor, which can consist of multiple image datasets and associated RT Structure Sets, and RT Dose objects, are properly accepted.
·        When the scenario starts a dialog will pop up where the user can select the desired scenario flow:
1. Send: CT-PET Hybrid image set 
2. Send: CT-MR Hybrid image set
Transactions sent to actor
RAD-4.8: Modality Images Stored
MMRO-4: Registered Structure Set Retrieval
MMRO-5: Registered Dose Retrieve
Expected transactions from actor
Not applicable
Used scenario dataset
The Hybrid dataset supplied with the installation of the IHE-RO MMRO II testtool.
