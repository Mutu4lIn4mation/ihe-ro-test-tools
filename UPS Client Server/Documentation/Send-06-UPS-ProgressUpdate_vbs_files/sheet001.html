Send UPS Progress Update
Instructions for use test scenarios
Description:
This test scenario sends a N-Set to the UPS server.
1. Make sure the system under test (SUT) is active and listening.
2. Start the scenario. Select the UPS to update and the test tool sends the N-Set request.
3. The following dialog wil be shown (Referenced Beam Number In Progress only by a treatment UPS).
4. With the Select N-Set template the dataset that is used for the N-Set can be selected.
5. Click OK to close this dialog.
6. Select the UPS to use and the test tool sends an N-Set request to that UPS.
7. The test tool automatically detects the N-Set reponse and will stop executing.
Expected transactions from actor:
        UPS-05: UPS Progress Update
In the respons a status (0000,0900) will be available, beneath a table with the meaning of the status codes. Meaning Code
Status The requested state change was performed 0
Success The UPS is already in the requested state of CANCELED B304
Warning The UPS is already in the requested state of COMPLETED B306
Refused: The UPS may no longer be updated C300
Failure Refused: The correct Transaction UID was not provided C301
Refused: The UPS is already IN PROGRESS C302
Refused: The UPS may only become SCHEDULED via N-CREATE, not N-SET or N-ACTION C303
Refused: The UPS has not met final state requirements for the requested state change C304
Specified SOP Instance UID does not exist or is not a UPS Instance managed by this SCP C307
Refused: The UPS is not yet in the "IN PROGRESS" state C310
