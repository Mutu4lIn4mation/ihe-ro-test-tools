Send UPS Subscribe Or Unsubscribe
Instructions for use test scenarios
Description:
This test scenario sends a N-Action to the UPS server.
1. Make sure the system under test (SUT) is active and listening.
2. Start the scenario. Select subscribe or unsubsribe in upcoming dialogbox.
  3. Select the UPS to use and the test tool sends an N-Action request for that UPS.
4. The test tool automatically detects the N-Action reponse and will stop executing.
Expected transactions from actor:
·        UPS-08: Subscribe to UPS
·        UPS-09: Unubscribe to UPS
In the respons a status (0000,0900) will be available, beneath a table with the meaning of the status codes.
Status  Meaning  Code 
Success The requested state change was performed 0
Warning The UPS is already in the requested state of CANCELED B304
The UPS is already in the requested state of COMPLETED B306
Failure Refused: The UPS may no longer be updated C300
Refused: The correct Transaction UID was not provided C301
Refused: The UPS is already IN PROGRESS C302
Refused: The UPS may only become SCHEDULED via N-CREATE, not N-SET or N-ACTION C303
Refused: The UPS has not met final state requirements for the requested state change C304
Specified SOP Instance UID does not exist or is not a UPS Instance managed by this SCP C307
Refused: The UPS is not yet in the "IN PROGRESS" state C310
