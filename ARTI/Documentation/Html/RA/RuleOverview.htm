Rule overview  
Ruletype Description
BeamLimitingDeviceSequenceRule checks how many device types there are in the RT Plan and of which type these are
BeamLimitingDeviceTypeRule Validates beam limiting device type data as described in the IHE technical framework. This is different among the beam types.
BeamModifierDetectionRule Reports if a beam modifier is detected in the dataset.
CompareRule compares an incoming dicommessage with a dicommessage in the reference dataset
CompatibilityRule verifies if the value in this attribute is a correct one, according to the value that is provided
ConditionalSequenceRule Applies another rule on the first item in the sequence if this sequence is present.
ConstantValueRule The value of an attribute is constant over more then one sequence item. The last item in the sequence should not be present.
ControlPointConstantValueRule verifies if value in control point sequence is not changed it may only exist in first control point sequence item. Otherwise if the value is changed every control point sequence item must have a value.
ControlPointOptionalValueRule Checks that an optional attribute in de control point sequence has the given value.
OptionalConstantValueRule The value of an optional attribute is constant over more then one sequence item. The last item in the sequence should not be present.
DateTimeRule Checks that the DateTime attribute has a value which is between 2 specified values.
DicomHasValueRule Checks if the attribute is present with a value in the dataset
DicomIsRequiredRule Checks the attribute is present in the dataset
DicomValueRule verifies if the value in this attribute is a correct one, according to the value that is provided
DifferentValueRule Checks if all attributes of a certain type in one sequence have different values. The last item in the sequence should not be present.
EmptySequenceRule Checks if sequence has no items
EmptyValueRule Checks if the attribute has no value
HasValueIfPresentRule Checks if the attribute has a value if present 
HasValueRule verifies if the value in this attribute is a not empty
IODReceivedRule Checks that atleast one object should be received.
IsRequiredRule Checks if the attribute or sequence is present.
MappingRule Checks if an attribute in a request dataset has the same value as defined in the
response dataset.
NotPresentRule Checks if the attribute or sequence is NOT present
NrOfSequenceItemsRule Checks that a sequence has the same number of sequence items indicated by the value of another attribute.
OptionalRule Reports if an attribute is present or not.
PIMTextValueRule Validates if the value of the Text Value (0040,A160) of Performed Procedure Information Module has one of the Beam Numbers of the RT Plan
ReferencedSequenceRule Validates that the correct reference to the required sequence is present.
RequiredConditionalRule When a conditional attribute is present and has a certain value, it checks if a required attribute is present.
SequenceLoopRule Applies another rule for each item in a sequence.
SequenceRangeRule Checks if the value multiplicity value of a sequence attribute is within the given range
TBDRule To Be Defined Rule. The Technical committee of IHE-RO has to define requirements
TreatmentDeliveryTypeRule Checks that the Treatment Delivery Type of the first Beam Task Sequence Item has the value Continuation and if the attributes: DosimeterUnit, StartMeterset and EndMeterset are present if the isContinuation parameter is set true
ValueEqualOrHigherRule Checks if the attribute has a value that is bigger then 1.
ValueInRangeRule Checks if the attribute is in the given range.
ValueListRule Checks if the attribute has the right value given in the specified list of values.
ValueRule Checks if the attribute has the right value.
